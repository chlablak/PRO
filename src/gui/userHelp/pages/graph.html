Graph

Declaring a new graph

There are many ways to declare a graph, from a simple empty graph:

g = { };

to a graph containing a certain number of elements:

g = {#3};

to more complex declarations with labels, weights and capacities. One important rule to note, is that the use of the # shortcut for multiple vertices must be the first element in the { } braces.

If you wish to declare a graph with vertices and edges, it can be done by including them between the { } braces:

g = {0,1,2,0--1,1--2};

or by adding them with the += operator;

g += (3);

for a vertex with the ID 3, or

g += (1--2);

for an edge joining the edges 1 and 2. There are two important rules adding elements to a graph:

  • When adding a vertex to a graph, all the previous IDs up the the current one must be already used (e.g. one can only add the vertex 3 if the vertices 0, 1 and 2 already exist in the graph). This also means vertices must be added in order when declaring or adding them to the graph. For instance g = {0,1,2,3} is correct, while g = {1,2,3,0} is not.
  • Edges can only be added between already existing vertices. For instance g = {0,1,2,1--2} is correct, while g = {0,1,2,2--3} and g = {0,1,1--2,2} are both incorrect.

Declaring a graph with vertices or edges

For now, it is only possible to declare a non-empty graph with anonymous edges and vertices, therefore:

g = {0,1,2};

is valid, while:

v = (0);

w = (1);

g = {v,w};

is not. This possibility will hopefully be implemented in the future. For now, you should declare an empty graph and add element with the += operator.
