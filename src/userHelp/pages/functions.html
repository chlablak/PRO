Functions

Calling functions

All graph transformations are done by passing a graph to a function. Once a function is called, the result can be assigned to an existing graph, the same graph or even a new graph. The notation is similar to many programming languages:

result = dijkstra(g, 1);

Note: Many functions return a Boolean type to inform you if the function or operation was successful or not.


Built-in functions

  • toString (a : T) : String returns a string describing the element a.
  • save (g : Graph) : Boolean saves the graph in the current directory.
  • load (file : String) : Graph loads a graph with file name from current directory.
  • type (a : T) : String returns the type of a.
  • er (V : Integer, p : float) : Graph generates a random graph with V vertices and a p probability of edges, according to Erdős–Rényi model.


GUI functions

  • draw (g : Graph) : Boolean draws g in a new window.
  • exportAsSvg (g : Graph) : Boolean exports the Graph in SVG format.
  • exportAsSvg (g : Graph, file : String) : Boolean exports the Graph in SVG format with file name.

Algorithms

  • bellmanFord (g : Graph, from : Integer) : Array applies the Bellman-Ford algorithms to g from vertex from. The Array returned contains the new graph at index 0 and the distance matrix at index 1.
  • bfs (g : Graph, from : Integer) : Array applies the breadth-first search algorithm to g from specified vertex. The returned Array contains the new graph at index 0 and the distance matrix at index 1.
  • cc (g : graph) : Array applies the connected component algorithms to g and returns an Array associating each vertex with its component. e.g.: [1,1,2,2,3]means vertices 0 and 1 are in the same component, 2 and 3 also while vertex 4 is alone.
  • detectCycle (g : Graph) : Graph; returns a cycle of the graph if there is one.
  • dfs (g : Graph, from : Integer) : Array applies the depth-first search algorithm to the graph and returns an Array with the resulting graph at index 0 and the visit order at index 1.
  • dijkstra (g : Graph, from : Integer) : Array applies Dijkstra's algorithm and returns an Array containing the new graph at index 0 and the distance matrix at index 1.
  • kruskal (g : Graph) : Graph applies Kruskal's algorithm and returns the resulting graph.
  • tarjan (g : Graph) : Array applies Tarjan's Algorithm to g (a directed Graph) and returns an Array containing the resulting graph at index 0 and an Array of connected component at index 1 (see bfs for an example).
  • topologicalSort (g : Graph) : Array applies a topological sort to g (which must be directed and not contain any cycles) and returns an Array with vertices in order of visit.
  • isConnected (g : Graph) : Boolean indicates if the graph is connected.
  • isDirected (g : Graph) : Boolean indicates if the graph is directed.
  • isEmpty (g : Graph) : Boolean indicates if the graph is empty (without edges).
  • isNegativeWeighted (g : Graph) : Boolean indicates if there are any negative weights.
  • isNull (g : Graph) : Boolean indicates if the graph is null (without any edge or vertex).
  • isPlanar (g : Graph) : Boolean indicates if the graph is planar.
  • isSimple (g : Graph) : Boolean indicates if the graph is simple (no doubled edges).
  • isStronglyConnected (g : Graph) : Boolean indicates if the graph is strongly connected (for directed graphs only).
  • isWeighted (g : Graph) : Boolean indicated if the graph has any weighted edges or vertices.


Parameter passing mode

All parameters in Graphy are passed by value. This means modifying a vertex after having added it to the graph will not change its properties in the graph, for instance if you write:

g = { };

v = (0::5); g += v; v = (0::3); the vertex with ID 0 in the graph will still have a weight of 5.
